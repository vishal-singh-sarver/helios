Installer Creation Guide

Build & Package Helios

Step-by-step instructions for creating production-ready installers for macOS, Windows, and Linux from a single codebase.

 macOS — DMG & PKG ⬜ Windows — NSIS EXE ⬤ Linux — AppImage & DEB
⚙

Prerequisites

Install these tools before running any build commands

Node.js
v22.0.0 or higher
npm
v10.0.0 or higher
Python
v3.10 or higher
Git
Any modern version
⚠️
Each platform must build on its native OS. Cross-compilation is not supported for Electron. Build macOS installers on macOS, Windows installers on Windows, and Linux installers on Linux.
1
Verify tool versions
node --version    # must be v22+
npm --version     # must be v10+
python3 --version # must be 3.10+
git --version
2
macOS — Install Xcode Command Line Tools
Required for native module compilation on macOS.
xcode-select --install
3
Linux — Install build essentials
Required for native module compilation and AppImage support.
sudo apt-get update
sudo apt-get install -y build-essential python3-dev makeself zenity
4
Clone the repository
git clone <repo-url>
cd helios_gui
5
Install Node dependencies
npm install
6
Set up Python virtual environment & backend
Only required if the sync-backend script does not create the venv automatically.
python3 -m venv backend-api/venv
source backend-api/venv/bin/activate   # Windows: backend-api\venv\Scripts\activate
pip install -U pip
pip install -r backend-api/requirements.txt
pip install pyinstaller
deactivate
🛠

Build the App

Compile TypeScript, React, and the backend binary before packaging

1
Compile everything
Runs sync-backend first (copies the backend binary), then compiles the main & renderer processes.
npm run build
ℹ️
Output lands in out/. Do not distribute files from out/ directly — use a packaging command instead.
2
Verify the build succeeded
No TypeScript errors should appear. Check that out/main/index.js and out/renderer/ exist.
ls out/main/index.js
ls out/renderer/


macOS Installer

Creates a DMG (drag-and-drop) and a PKG (professional installer)

1
Ensure you are on a Mac
macOS packaging requires running on native macOS hardware or a macOS VM.
2
Confirm the macOS backend binary exists
The sync script copies the compiled Python binary here during npm run build.
ls resources/backend/mac/heliosgui_backend
3
Run the macOS packaging command
npm run package:mac
ℹ️
This internally runs npm run build first, then calls electron-builder --mac.
4
Locate the output installers
ls dist/
# Helios-1.0.0.dmg   (~135 MB)
# Helios-1.0.0.pkg   (~135 MB)
5
Test the DMG installer
Double-click Helios-1.0.0.dmg, drag Helios.app into Applications, then launch.
⚠️
Unsigned builds show a Gatekeeper warning. Right-click the app and choose Open, or run: xattr -d com.apple.quarantine /Applications/Helios.app
6
Test the PKG installer
Double-click Helios-1.0.0.pkg and follow the guided installer wizard (Welcome → License → Location → Auth → Progress → Done).
7
(Production only) Code-sign and notarize
Add your Apple Developer identity to electron-builder.yml before shipping publicly.
# electron-builder.yml
mac:
  identity: 'Developer ID Application: Your Company (TEAM_ID)'
  sign: true
⬜

Windows Installer

Creates an NSIS-based setup EXE with Start Menu and Desktop shortcuts

1
Ensure you are on Windows
Windows packaging requires native Windows tools. Use Windows 10 or later.
2
Place the Windows backend binary
Compile or obtain the Windows binary and place it here:
resources/backend/win/heliosgui_backend.exe
3
Activate the Python venv (Windows syntax)
python -m venv backend-api\venv
backend-api\venv\Scripts\activate
pip install -U pip
pip install -r backend-api\requirements.txt
pip install pyinstaller
deactivate
4
Run the Windows packaging command
npm run package:win
5
Locate the output installer
dir dist\
# "Helios Setup 1.0.0.exe"  (~150 MB)
6
Test the installer end-to-end
Double-click the .exe. Walk through: UAC prompt → Welcome → License (check "I Agree") → Install path → Install → Finish. The app should launch automatically.
ℹ️
Shortcuts are created on the Desktop and in the Start Menu automatically.
7
(Production only) Code-sign for SmartScreen
# electron-builder.yml
win:
  certificateFile: path/to/cert.pfx
  certificatePassword: ${CERTIFICATE_PASSWORD}
Set CERTIFICATE_PASSWORD as an environment variable before building.
⬤

Linux Installer

Creates an AppImage (portable) and a DEB package (system integration)

1
Ensure you are on Linux
Ubuntu 18.04+ or any modern Debian-based distro is recommended.
2
Place the Linux backend binary
Compile or obtain the Linux binary and place it here (no extension needed):
resources/backend/linux/heliosgui_backend
3
Set up the Python virtual environment
python3 -m venv backend-api/venv
source backend-api/venv/bin/activate
pip install -U pip
pip install -r backend-api/requirements.txt
pip install pyinstaller
deactivate
4
Fix Electron sandbox permissions (one-time)
Required if you plan to test in dev mode. Not needed for packaging only.
sudo chown root:root node_modules/electron/dist/chrome-sandbox
sudo chmod 4755 node_modules/electron/dist/chrome-sandbox
5
Run the Linux packaging command
npm run package:linux
6
Locate the output packages
ls dist/
# Helios-1.0.0.AppImage  (~150 MB)
# helios-1.0.0.deb       (~140 MB)
7
Test the AppImage
chmod +x dist/Helios-1.0.0.AppImage
./dist/Helios-1.0.0.AppImage
8
Test the DEB package
sudo dpkg -i dist/helios-1.0.0.deb
helios   # or: /opt/helios/Helios
ℹ️
To uninstall: sudo apt remove helios. User data in ~/.config/Helios/ is preserved.
9
(Optional) Build the custom .run installer
Requires makeself and zenity (already installed in step 3 of prerequisites).
./linux-installer/build-linux-run.sh
📦

Output Files Reference

All distributable files land in the dist/ directory

File Platform Type Size (est.) End-user action
Helios-1.0.0.dmg macOS Drag-and-drop disk image ~135 MB Drag app to Applications
Helios-1.0.0.pkg macOS Professional installer wizard ~135 MB Run installer wizard
Helios Setup 1.0.0.exe Windows NSIS installer ~150 MB Double-click & follow wizard
Helios-1.0.0.AppImage Linux Portable executable ~150 MB chmod +x then run
helios-1.0.0.deb Linux Debian package ~140 MB sudo dpkg -i
ℹ️
The large size is expected — Electron's runtime (~120 MB) is bundled inside every installer. To reduce size: remove unused node_modules dependencies and ensure tree-shaking is enabled.

Backend Integration

How the Python FastAPI backend is compiled and bundled into the installer

1
Backend resource structure
Binaries must be placed in platform-specific subdirectories before packaging:
resources/backend/
├── mac/   heliosgui_backend       (no extension)
├── win/   heliosgui_backend.exe
└── linux/ heliosgui_backend       (no extension)
2
How the sync script works
scripts/sync-backend.js runs automatically during npm run build. It copies the compiled PyInstaller binary into the correct resources/backend/ subdirectory and reports file sizes. Missing win/linux binaries produce warnings, not errors.
3
How the backend starts at runtime
src/main/backend-manager.ts detects the platform at startup, resolves the binary path (packaged vs. dev mode), spawns the process on port 8008, and shuts it down gracefully when the app quits (SIGTERM → 5s → SIGKILL).
4
User data paths per platform
Windows : %APPDATA%\Helios\
macOS   : ~/Library/Application Support/Helios/
Linux   : ~/.config/Helios/
🔍

Troubleshooting

Common issues and their fixes

⚠
Build fails: icon or license file not found
ls build/icon.ico
ls build/license.txt
Both files must exist before packaging.
⚠
Backend won't start after install
Check the log files for the specific error:
# macOS
cat ~/Library/Application\ Support/Helios/logs/backend.log

# Linux
cat ~/.config/Helios/logs/backend.log

# Windows
type %APPDATA%\Helios\logs\backend.log
⚠
Port 8008 already in use
# macOS / Linux
lsof -i :8008 | awk 'NR!=1 {print $2}' | xargs kill -9

# Windows
netstat -ano | findstr :8008
taskkill /PID <PID> /F
⚠
Complete clean rebuild
npm run dist:clean-all    # removes dist/, out/, node_modules/
npm install
npm run build
⚠
Clean all caches and app data (macOS)
sudo rm -rf /Applications/Helios.app
rm -rf ~/Library/Application\ Support/Helios
rm -rf ~/Library/Caches/Helios
rm -rf ~/Library/Preferences/com.navyug.helios.plist
rm -rf ~/.cache/electron
rm -rf ~/.cache/electron-builder
⚠
Reset all build artifacts
rm -rf dist out
rm -rf backend-api/build backend-api/dist
rm -rf linux-installer/payload/Helios
rm -f  resources/backend/mac/heliosgui_backend
rm -rf resources/backend/linux/heliosgui_backend
